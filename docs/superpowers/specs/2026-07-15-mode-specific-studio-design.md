# 分类化图像创作工作台设计

日期：2026-07-15

## 目标

将创作工作台从“所有分类共用文生图设置”调整为“按创作类型展示对应设置”，同时把提示词模板移动到右侧预览区，并支持生成结果一键进入局部编辑。

## 分类与设置

- **文生图**：模型、比例、数量、分辨率、提示词。
- **图生图**：参考图、比例、数量、分辨率、风格模板、提示词、参考图强度与构图保持预留控件。
- **局部编辑**：原图、遮罩画笔、画笔大小、撤销、重做、清空、修改提示词；不展示分辨率和通用比例设置。
- **图片编辑**：去背景、换背景、换衣服、换脸、加文字五个编辑类型；根据编辑类型展示原图、参考图或文字输入，不展示分辨率设置。
- **超分**：2×、4×、图片变体、老照片修复、人脸增强等超分选项；以原图和提示词为核心。
- **批量一致性**：角色参考图、数量、角色一致性、场景提示词、构图变化。

模式分类收敛为六类：文生图、图生图、局部编辑、图片编辑、超分、批量一致性。原“换背景”并入“图片编辑”，不再重复占用一级分类。

## 提示词模板

提示词模板从左侧参数区移到右侧预览区，作为独立卡片展示。点击模板后只填充当前分类的提示词输入框，不直接提交任务。模板内容按当前分类过滤，文生图不再在左侧显示模板卡片。

## 结果图回流局部编辑

单图和多图结果卡片在悬浮、键盘聚焦或移动端触控时显示“局部编辑”。点击后：

1. 切换当前分类为“局部编辑”；
2. 将结果 URL 作为当前源图保存在前端状态；
3. 打开现有遮罩编辑器；
4. 不要求用户重新选择或上传结果图。

源图 URL 保留为 `url`，遮罩继续由现有画笔生成。当前后端尚未提供专用局部编辑语义，前端只完成工作台回流和请求结构准备，不宣称已经具备真正的后端遮罩处理。

## 文本选择与交互

商业前端正文、提示词、模板说明和结果状态文字允许浏览器选择复制；按钮、图标、图片拖拽画布和控制项保持不可选。所有新增按钮有可见焦点、键盘操作和移动端至少 44px 触控区域。

## 接口边界

继续使用现有 `POST /api/image/tasks` 请求字段。新增的专用控件先保存在当前工作台状态，并通过提示词上下文表达可兼容的意图；不新增前端伪造 API，不修改后端契约。后续专用 API 接入时，只替换各模式的请求映射。

## 验收

- 六个分类的左侧控件内容明显不同。
- 图生图仍包含分辨率、风格模板和提示词。
- 图片编辑不显示分辨率控件，且有五个编辑子类型。
- 超分显示 2×/4×、图片变体、老照片修复、人脸增强。
- 模板只在右侧出现，点击后填入当前分类提示词。
- 生成结果出现局部编辑按钮，点击后进入局部编辑并带入结果 URL。
- 正文和提示词可复制，交互控件不因拖拽预览而阻断文本选择。
- 1920×1080、1366×768、390×844 无横向溢出。
