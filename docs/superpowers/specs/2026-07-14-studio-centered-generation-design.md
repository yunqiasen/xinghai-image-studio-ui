# 图像创作页居中工作台与生成结果设计

## 已确认方向

采用用户确认的第一张居中版视觉稿。桌面工作区最大宽度 1240px，水平居中；左侧固定约 480px，右侧使用剩余空间。1366px 浏览器保留约 60px 两侧留白，1600px 浏览器保留约 180px 两侧留白。手机端改为上下排列。

## 功能边界

保留现有七种创作模式、上传素材、模型、七种宽高比、生成张数、1K/2K/4K、提示词模板、提示词、积分估算和原图打开。前端只调用后端已发布契约 `POST /api/image/generate`，不新增接口、不修改后端。

## 左侧结构

左侧由固定标题、可用高度内的主体区和固定底部提交栏组成。主体区分为竖排模式栏与参数滚动区。底部显示模式、张数、分辨率、预计积分和生成按钮，滚动参数时始终可见。

## 右侧状态

右侧统一分为标题栏、画布区和状态栏：

1. 空闲：比例感知空画布，提示等待生成。
2. 生成中：点击按钮后立即渲染与生成张数一致的加载占位，显示旋转动画和等待时间；同步 API 请求在后台等待。
3. 单图结果：完整显示、不裁切，提供缩放、适应画布和打开原图。
4. 多图结果：两张采用双列；三张和四张采用 2x2 网格。每张保持所选宽高比并可打开原图。
5. 失败：右侧显示中文错误状态，保留当前参数以便重新生成。

## API 数据流

`StudioPage.submit()` 校验登录与素材后设置生成中状态，调用 `submitImageGeneration()`。客户端向 `/api/image/generate` 发送 Cookie Session 和契约字段。成功后把 `imageUrls` 数组交给右侧预览；失败优先显示后端 `message`，其次 `error` 和 HTTP 状态。后端返回的新用户积分继续写回当前会话。

## 验证

自动化测试覆盖 API 错误归一化、右侧加载状态、单图/多图输出和居中布局常量。完成后执行 `npm run lint`、`npm run test`、`npm run build`、`git diff --check`，再更新 `docs/API_INTEGRATION.md` 与前端 README 中的创作页行为。
